# AKQuant V0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a runnable V0.1 A-share backtesting core with deterministic fixture data, A-share rule enforcement, dual moving average strategy, metrics, report export, and CLI entry point.

**Architecture:** Implement a lightweight Python package with separated modules for models, data loading, market rules, strategy, engine, analytics, and reporting. The first implementation uses standard-library CSV/Markdown output so it can run without network or heavy dependencies; AKShare integration is represented as a future adapter boundary from the PRD.

**Tech Stack:** Python 3.11+, dataclasses, csv/json/argparse/pathlib, pytest.

---

## File Structure

- `pyproject.toml`: package metadata and pytest configuration.
- `README.md`: local usage and scope.
- `akquant/__init__.py`: package exports.
- `akquant/models.py`: bars, orders, fills, positions, configs, results.
- `akquant/data.py`: CSV fixture loader and data validation.
- `akquant/rules.py`: T+1, board-lot, cash/position, cost and limit checks.
- `akquant/strategies.py`: dual moving average strategy.
- `akquant/engine.py`: daily event loop and portfolio accounting.
- `akquant/analytics.py`: performance and trade metrics.
- `akquant/reporting.py`: Markdown report export.
- `akquant/cli.py`: command-line backtest runner.
- `tests/fixtures/sample_bars.csv`: deterministic daily bar data.
- `tests/test_rules.py`: market rule behavior.
- `tests/test_engine.py`: end-to-end backtest behavior.
- `tests/test_analytics_reporting.py`: metrics and report behavior.
